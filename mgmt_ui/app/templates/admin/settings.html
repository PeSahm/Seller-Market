Bot runtime / endpoints

Values that used to be hardcoded in the bot image — broker / market-data hosts, the exir buy-fee fallback, the auto-sell timing. Editing one here renders it into every stack's config.ini [runtime] section and pushes it to all stacks in seconds on Save — no CI, no image rebuild, no container recreate. Each default equals the bot's current built-in value, so nothing changes until you edit it. Requires the runtime-aware bot image fleet-wide (older images ignore [runtime] and keep their built-ins).

Base domain for ephoenix-family brokers (api-<broker>.<domain>, identity, backoffice, …).

Market-hours gate (HH:MM-HH:MM); hot-reloads.

Seconds the buy-queue must stay ≤ threshold before a sell fires; hot-reloads.

Advanced runtime overrides (escape hatch)

One bot_rt_<key> = value per line for the rare case a single API path moves. Supported keys the bot reads: full-URL endpoint overrides bot_rt_endpoint_<broker>_<name> (e.g. bot_rt_endpoint_ib_order), exir paths bot_rt_exir_path_<name>, RLC bot_rt_rlc_base_url / bot_rt_rlc_market_host / bot_rt_rlc_path_* / bot_rt_rlc_ws_scheme / bot_rt_rlc_ws_path. Remove a line to clear that override (the bot falls back to its built-in).
